P – Parse

What it is: turn raw inputs (PDFs, docs, web pages, tables) into clean, machine-usable chunks.

How: format detection, safe text extraction, structure recovery, light normalization, and metadata capture.

Why: clean parsing prevents hallucinations downstream and raises recall.

I – Index

What it is: build searchable representations for fast, accurate look‑ups.

How: hybrid BM25 + vector embeddings, fielded metadata, time/role filters, and periodic refresh.

Why: a thoughtful index trades brute force for relevance and speed.

R – Retrieval

What it is: find the few most useful passages for a given task and user intent.

How: query rewriting/expansion (HyDE), reranking, diversity control, and guardrails for scope.

Why: high‑quality retrieval narrows the model’s context to what truly matters.

AG – Augmented
Generation

What it is: compose answers by blending retrieved facts, tools, and reasoning.

How: grounded prompts, function/tool calls, multi‑step plans, and traceable citations.

Why: makes outputs useful, verifiable, and action‑ready.

E – Evaluation

What it is: a continuous quality loop that spans the whole pipeline.

How: offline tests and online signals: task success, answerability, faithfulness, latency, cost, and safety. Run AB tests, spot‑check transcripts, compare versions, and feed regressions back into P, I, R, and AG.

Why: without measurement, improvement is guesswork. With it, PIRAGE learns over time.
